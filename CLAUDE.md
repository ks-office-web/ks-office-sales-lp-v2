# K's Office — プロジェクト規約

## プロジェクト概要
プロジェクト名: **K's Office**
種別: セールスLP（ランディングページ）

---

## デザイン規約

### フォント
- **Noto Sans JP** のみ使用
- 使用ウェイト: **700・900のみ**（他のウェイト禁止）

### カラーパレット
使用できる色は以下の4色のみ:

| 役割 | カラー | 値 |
|------|--------|----|
| メイン | ネイビー | `#1a2a5e` |
| 背景 | オフホワイト | `#f4f1ea` |
| 背景・テキスト | 白 | `#ffffff` |
| テキスト | ほぼ黒 | `#111111`（相当） |

**禁止色: 黄色・赤（いかなるトーン・用途でも使用禁止）**

### アクセントカラー
- **LINEグリーン（`#06C755`）のみ**アクセントとして使用可
- 他のアクセントカラー追加不可

---

## コピー（テキスト）規約

- **コピー文言は絶対に勝手に変更しない**
- キャッチコピー・ボディコピー・CTAテキスト・注釈など、すべてのテキストが対象
- 変更が必要な場合は必ずユーザーに確認し、承認を得てから反映する

---

## 作業フロー規約

### LP作り込み前の確認
- デザイン・レイアウト・構成など、LP本体に大きく手を入れる前は**必ず事前確認を取る**
- 「これから〇〇を変更します」と明示し、承認後に着手する

### 作業後のコミット・プッシュ
- 作業完了後は以下を**自動で実行する**:
  ```
  git add .
  git commit -m "<適切なコミットメッセージ>"
  git push
  ```
- コミットメッセージは変更内容を端的に表す日本語または英語で記述する

---

## 禁止事項まとめ

1. 黄色・赤系カラーの使用
2. Noto Sans JP 700/900以外のフォントウェイト使用
3. コピー文言の無断変更
4. LP作り込み作業の無断着手
