## デザイン規約

### フォント
- **Noto Sans JP** のみ使用
- 使用ウェイト: **700・900のみ**（他のウェイト禁止）

### カラーパレット
使用できる色は以下の4色のみ:

| 役割 | カラー | 値 |
|------|--------|----|
| メイン | ネイビー | `#1a2a5e` |
| 背景 | オフホワイト | `#f4f1ea` |
| 背景・テキスト | 白 | `#ffffff` |
| テキスト | ほぼ黒 | `#111111`（相当） |

**禁止色: 黄色・赤（いかなるトーン・用途でも使用禁止）**

### アクセントカラー
- **LINEグリーン（`#06C755`）のみ**アクセントとして使用可
- 他のアクセントカラー追加不可

### レイアウト
- 背景はオフホワイト／白の**交互サンドイッチ構造**

### 文字サイズ
| 用途 | サイズ |
|------|--------|
| 見出し | 44〜72px |
| 価格 | 148px |
| 本文 | 16px |

### 画像
- 写真は**base64埋め込み必須**（外部ファイル参照禁止）

## コピー（テキスト）規約

- **コピー文言は絶対に勝手に変更しない**
- キャッチコピー・ボディコピー・CTAテキスト・注釈など、すべてのテキストが対象
- 変更が必要な場合は必ずユーザーに確認し、承認を得てから反映する

---

## 言葉遣い規約

- **「シンプル」という表現は禁止**
- 職人向けの説明では**「LP」「Web」という言葉を使わない**
  - 代わりに → **「名刺に付けられる仕事紹介ページ」** と表現する

## 作業フロー規約

### LP作り込み前の確認
- デザイン・レイアウト・構成など、LP本体に大きく手を入れる前は**必ず事前確認を取る**
- 「これから〇〇を変更します」と明示し、承認後に着手する

### 作業後のコミット・プッシュ
- 作業完了後は以下を**自動で実行する**:
  ```
  git add . && git commit -m "内容" && git push
  ```
- コミットメッセージは変更内容を端的に表す日本語または英語で記述する

---

## 禁止事項まとめ

1. 黄色・赤系カラーの使用
2. Noto Sans JP 700/900以外のフォントウェイト使用
3. コピー文言の無断変更
4. LP作り込み作業の無断着手
5. 「シンプル」という表現の使用
6. 職人向け説明での「LP」「Web」という言葉の使用
